# DONS AI Agent Stack Guide

## Overview

The DONS AI Agent Stack is an optional enhancement that adds three intelligent agents to your migrated infrastructure, powered by DigitalOcean Gradient™ AI.

## The Three AI Agents

### 1. Cloud Migration Architect Agent 🏗️

**Purpose**: Intelligent migration planning and risk analysis

**Capabilities**:
- Analyzes AWS infrastructure and generates detailed migration plans
- Identifies risks and provides mitigation strategies
- Determines optimal deployment order based on dependencies
- Estimates migration duration and costs
- Generates rollback procedures

**Technology**:
- Gradient AI (llama3-8b-instruct) for plan generation
- RAG with OpenSearch for migration best practices
- Knowledge Base with migration documentation

**API Endpoints**:
```
POST /api/generate-plan
GET /api/risks
GET /api/deployment-order
```

**Example Usage**:
```python
import requests

response = requests.post(
    "https://migration-architect-agent.ondigitalocean.app/api/generate-plan",
    json={
        "aws_resources": [...],
        "do_resources": [...]
    }
)

plan = response.json()
print(f"Migration steps: {len(plan['deployment_steps'])}")
print(f"Estimated duration: {plan['duration_estimate']['formatted']}")
```

### 2. DevOps Agent 🔧

**Purpose**: Autonomous infrastructure monitoring and self-healing

**Capabilities**:
- Monitors resource usage (CPU, memory, disk, database storage)
- Detects threshold violations (>80% usage)
- Automatically generates Terraform fixes
- Creates GitHub PRs with infrastructure updates
- Sends alerts via webhooks
- Tracks self-healing action history

**Technology**:
- Gradient AI for analyzing metrics and generating fixes
- DigitalOcean Monitoring API for metrics
- GitHub API for PR creation
- Background worker for continuous monitoring

**API Endpoints**:
```
GET /api/alerts
POST /api/heal
GET /api/self-healing-actions
GET /api/metrics
```

**Self-Healing Example**:
When database storage reaches 80%, the agent:
1. Detects the threshold violation
2. Generates Terraform code to increase storage
3. Creates a GitHub PR with the fix
4. Notifies team via webhook
5. Applies fix after PR approval

**Example PR**:
```hcl
# Auto-generated by DONS DevOps Agent
# Issue: Database storage at 82% capacity

resource "digitalocean_database_cluster" "app_db" {
  name       = "app_db"
  engine     = "mysql"
  version    = "8"
  size       = "db-s-1vcpu-1gb"
  region     = "nyc1"
  node_count = 1
  
  # Increased from 20480 to 30720 (50% increase)
  storage_size_mib = 30720
}
```

## Infrastructure Components

**Purpose**: Vector storage for RAG (Retrieval-Augmented Generation)

**Configuration**:
```hcl

  name       = "rag-vector-store"

  version    = "2"

```

**Features**:
- Vector similarity search with kNN
- Full-text search capabilities
- Automatic storage autoscaling
- High availability with multi-node clusters

**Cost**: $60-720/month (depending on size)

**Configuration**:

## Deployment Options

### Option 1: Full Stack (Recommended for Production)

Deploy all three agents with full RAG infrastructure:

```bash
# Enable AI stack in Terraform
terraform apply -var="enable_ai_stack=true"
```

**Use when**:
- Production deployment
- Need all AI capabilities
- Budget allows $100-160/month

### Option 2: Selective Agents

Deploy only specific agents:

```bash
# Deploy only DevOps agent for monitoring
terraform apply -var="enable_devops_agent=true"
```

**Use when**:
- Testing specific features
- Budget constraints
- Gradual rollout

### Option 3: Development Mode

Skip AI stack during development:

```bash
# Disable AI stack
terraform apply -var="enable_ai_stack=false"
```

**Use when**:
- Local development
- Testing base infrastructure
- Cost optimization

## Setup Instructions

### 1. Prerequisites

```bash
# Get Gradient AI API key
# Visit: https://cloud.digitalocean.com/gradient-ai

# Get GitHub token for self-healing PRs
# Visit: https://github.com/settings/tokens
# Scopes needed: repo, workflow
```

### 2. Configure Variables

Create `terraform.tfvars`:

```hcl
# Required
do_token          = "dop_v1_..."
gradient_api_key  = "sk-do-..."
github_token      = "ghp_..."

# Optional
enable_ai_stack   = true
opensearch_size   = "db-s-2vcpu-4gb"
github_repo       = "your-org/your-repo"
github_branch     = "main"
```

### 3. Deploy

```bash
terraform init
terraform plan
terraform apply
```

### 4. Verify Deployment

```bash
# Get agent URLs
terraform output ai_agents

# Test Migration Architect
curl https://migration-architect-agent-xxxxx.ondigitalocean.app/health

# Test DevOps Agent
curl https://devops-agent-xxxxx.ondigitalocean.app/health

# Test AI Enablement Agent
curl https://ai-enablement-agent-xxxxx.ondigitalocean.app/health
```

### 5. Upload Knowledge Base Documents

```bash
# Upload migration guides
aws s3 cp docs/ s3://dons-knowledge-base/ --recursive \
  --endpoint-url=https://nyc3.digitaloceanspaces.com \
  --profile digitalocean
```

## Integration with DONS Platform

The AI agents are automatically integrated with the DONS web platform:


## Troubleshooting

### Agent Not Starting

**Symptoms**: App Platform shows "Error" status

**Solutions**:
1. Check Gradient AI API key is valid
2. Verify GitHub repository is accessible
3. Check environment variables are set
4. Review app logs for errors

```bash
doctl apps logs <app-id> --type=run
```

### OpenSearch Connection Failed

**Symptoms**: Agents can't connect to OpenSearch

**Solutions**:
1. Verify OpenSearch cluster is active
2. Check firewall rules
3. Verify credentials in environment variables
4. Test connection manually:

```bash
curl -u user:password https://opensearch-host:25060
```

### Knowledge Base Not Working

**Symptoms**: RAG queries return no results

**Solutions**:
1. Verify documents are uploaded to Spaces bucket
2. Check documents are indexed in OpenSearch
3. Verify Spaces credentials
4. Re-index documents:

```bash
python scripts/reindex_knowledge_base.py
```

### Self-Healing PRs Not Created

**Symptoms**: DevOps agent detects issues but doesn't create PRs

**Solutions**:
1. Verify GitHub token has correct scopes (repo, workflow)
2. Check GitHub repository exists and is accessible
3. Verify agent has write permissions
4. Check GitHub API rate limits

## Best Practices

### 1. Knowledge Base Management

- **Organize documents** by category (migration, troubleshooting, best practices)
- **Use versioning** to track document changes
- **Update regularly** with new learnings
- **Include metadata** (tags, categories) for better search

### 2. Self-Healing Configuration

- **Set appropriate thresholds** (80% is recommended)
- **Test PR creation** in a test repository first
- **Review PRs** before merging (don't auto-merge)
- **Monitor false positives** and adjust thresholds

### 3. Cost Optimization

- **Start small** with 1-node OpenSearch cluster
- **Scale up** based on actual usage
- **Use development mode** for testing
- **Monitor costs** in DigitalOcean billing

### 4. Security

- **Rotate API keys** regularly
- **Use secrets** for sensitive values
- **Restrict API access** with IP allowlists
- **Enable audit logging** for compliance
